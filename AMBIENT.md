# アンビエント変換

## アンビエント変換とは

Sumibiの通常の変換はCtrl+Jで実行しますが、アンビエント変換は助詞や句読点の入力をトリガーに、意識することなく自然に日本語変換が行われる機能です。

## 動作例

### 助詞+スペースでの自動変換
- `watashiwa ` → `私は`
- `nihonga ` → `日本が`
- `tokyode ` → `東京で`

### 句読点での自動変換
- `wakarimashita.` → `わかりました。`
- `sorekara,` → `それから、`
- `nandesuka?` → `なんですか？`
- `karini,` →`仮に、`

## 設定方法

init.elに以下を追加してください。

```elisp
(setq sumibi-ambient-enable t)
```

## 英文の自動判定

英文を入力中に誤って日本語変換が発動しないよう、テキスト中の英単語の割合が80%以上の場合は自動変換をスキップします。固有名詞（Claude Code等）、短縮形（don't, it's等）、句読点付き単語も正しく英語として認識します。

## 除外される環境

以下の環境では、アンビエント変換は自動的に無効になります。

| 環境 | 理由 |
|---|---|
| シェルバッファ（shell-mode） | コマンドやファイルパスの誤変換を防ぐ |
| GPG暗号化ファイル（.gpg） | 機密データがLLM APIに送信されるのを防ぐ |
| ミニバッファ（Find File:等） | ファイル名入力の誤変換を防ぐ |

### 除外環境のカスタマイズ

カスタマイズ変数を使って、無効化したいメジャーモードやファイル拡張子を追加することもできます。

```elisp
;; 除外するメジャーモードを追加
(setq sumibi-ambient-exclude-modes
      '(shell-mode emacs-lisp-mode))

;; 除外するファイル拡張子を追加
(setq sumibi-ambient-exclude-file-extensions
      '("gpg" "secret"))
```
